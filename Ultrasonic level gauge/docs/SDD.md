# SDD — Ultrasonic level gauge

## Цель
Уровнемер жидкости: Arduino Nano 3 + JSN-SR04T + OLED 0.96" 128×64 I2C.  
На дисплее — открытая ёмкость со шкалой объёма; объём считается по высоте столба воды.

## Железо
| Узел | Модель / примечание |
|------|---------------------|
| МК | Arduino Nano 3 (ATmega328) |
| Датчик | JSN-SR04T (логика как HC-SR04; корпус водозащищённый) |
| Дисплей | OLED 0.96" 128×64 I2C (SSD1306, адрес обычно `0x3C`) |
| Среда проверки (сейчас) | Proteus (виртуальные соединения) |

## Пины (Nano)
| Сигнал | Пин | Примечание |
|--------|-----|------------|
| OLED SDA | A4 | I2C |
| OLED SCL | A5 | I2C |
| OLED RES | D4 | сброс дисплея |
| Датчик TRIG | D9 | режим TRIG/ECHO |
| Датчик ECHO | D10 | режим TRIG/ECHO |
| Датчик UART RX | D8 (план) | Nano ← TX датчика; раскомментировать в скетче |
| Датчик UART TX | D7 (план) | Nano → RX датчика; раскомментировать в скетче |
| Питание модулей | +5V / GND | общий GND обязателен |

## Режимы датчика
В скетче переключение через `#define`:

- **`SENSOR_MODE_TRIG_ECHO`** — активен сейчас; тест в Proteus (D9/D10).
- **`SENSOR_MODE_UART`** — закомментирован; при смене режима раскомментировать блок SoftwareSerial и закомментировать TRIG/ECHO.

Оба варианта держим в одном скетче с пометками, что комментировать.

## Параметры ёмкости (калибровка)
Квадратное сечение → объём пропорционален высоте воды.

| Параметр | Значение | Где править |
|----------|----------|-------------|
| Полный объём | `12.0` м³ | `TANK_CAPACITY_M3` |
| Высота дно → горло | `1.8` м | `TANK_HEIGHT_M` ← калибровка |
| Площадь сечения | `V / H` ≈ 6.667 м² | `TANK_AREA_M2` (считается) |

Формулы:
- `volume_m3 = level_m / TANK_HEIGHT_M * TANK_CAPACITY_M3`
- функции: `volumeFromLevel_m3()`, `levelFromVolume_m()`

## UI (OLED)
- Ёмкость без верхней крышки (левая стенка, дно, правая стенка).
- Заливка воды снизу вверх по объёму.
- Справа шкала делений: **0, 4, 8, 12** (м³); 0 — у дна, 12 — у горла.
